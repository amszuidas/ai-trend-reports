## 今日 Feature 趋势摘要

1. **AI/Agent 占据 21 个榜单席位中的 16 个，并包揽前 6 名。** 热点从 Agent 教材、响应风格 Skill、Coding Agent Harness，延伸到代码上下文、专业桌面操作、结构化输出、本地模型选型和实时世界数据。
2. **Agent 的价值越来越取决于“拿到什么上下文”。** Code Review Graph 用结构关系缩代码上下文，Wigolo 提供可追溯 Web 证据与本地缓存，World Monitor 暴露实时世界数据，OpenSEO 把 SEO/GSC 数据变成 MCP；模型本身不再是唯一焦点。
3. **Skill 正在成为可安装、可审计的专业工作流单元。** `i-have-adhd` 交付响应规范，`text-to-cad` 交付 CAD/机器人/制造流程，OpenSEO 交付 7 个 SEO 工作流，Ontology Playground 也用 Skills 维护本体内容；但 Skill 是指令资产，不等于新模型或可靠 Runtime。
4. **多模型资源控制面持续升温。** OmniRoute 聚合订阅、Key、本地模型、配额、预算、路由和压缩；llmfit 则在下载前回答硬件适配、量化、上下文和性能预估。两者共同反映“模型太多、额度太碎、硬件条件太复杂”的现实痛点。
5. **专业应用的 Agent 接口比通用聊天更吸睛。** TradingView MCP 操作用户已登录的图表/Pine 工作区，World Monitor 向 Agent 提供地缘/市场/灾害数据，AstrBot 把 Agent 送进多种 IM，pi-web 给现有 Pi 会话增加可视工作台。
6. **“Local-first / Free / No keys”是最强传播文案，也是最需要拆解的承诺。** 本地缓存不代表搜索请求不外发，本地 AI 不代表世界数据离线，开源网关不代表上游模型免费，自托管 OpenSEO 仍依赖付费 DataForSEO。
7. **热度与成熟度明显错位。** 今日同时出现正式 1.x SDK、0.x 高频迭代产品、Public Beta、Preview、无 Release 的 Skill/研究仓库和需要外部桌面/硬件的工具；可演示、Stars 暴增与生产可靠性必须分开看。

## Feature 深度卡片

### 1. [World Monitor](https://github.com/koala73/worldmonitor)

**一句话定位：** 面向记者、OSINT/风险、供应链和 Agent 开发者，把新闻、冲突、航运、灾害、基础设施、宏观与市场信号汇成可视情报工作台，并通过 MCP/API 供 Agent 查询；它是研究起点，不是经过人工逐条核验的专业情报终端。[功能总览](https://www.worldmonitor.app/docs/features)

**典型场景：** 突发事件跨源态势感知；国家/区域风险初筛；供应链航线压力测试；个性化新闻与告警；给 Agent 补实时世界上下文。

| Feature | 能做什么 → 用户价值 → 适用场景 | 状态 / 边界 |
|---|---|---|
| **56 类双引擎地图** | 在 3D/平面地图叠加冲突、军民航、海缆、管道、AI 数据中心、灾害和网络中断 → 发现共址/连锁风险 → 战区与资产风险 | Free；部分实时层需 Key/relay，[图层](https://www.worldmonitor.app/docs/features#data-layers) |
| **500+ feeds + World Brief** | 聚类去重并生成带来源简报 → 少读重复信息且可回查 → 晨报/突发概览 | 公共流可用；AI 可走本地或云模型，[方法](https://www.worldmonitor.app/docs/methodology/news-digest-and-briefing) |
| **跨流关联 / Breaking Alert** | 关联新闻、抗议、军机、舰船、网络等信号 → 优先看多源同时升温对象 → 值班监控 | 核心面板可用；外部通知 Pro |
| **本地 AI + Headline Memory** | 接 Ollama/LM Studio，在浏览器本地做标题向量检索/分类/摘要 fallback → 降低敏感主题外发 → 私密研究 | 需配置；Memory 默认关闭、最多 5,000 向量，移动端禁用，[AI](https://www.worldmonitor.app/docs/ai-intelligence) |
| **国家 Brief + CII/CRI** | 汇总事件并区分短期压力与结构韧性 → 快速筛国家 → 风险/政策研究 | CII 仅 31 国，CRI 为自定义指标，不能当评级结论 |
| **Route Explorer** | 按起终点、HS2、货型展示海峡暴露、绕行和依赖 → 把地缘风险落到具体路线 → 供应链/商品研究 | Pro，[文档](https://www.worldmonitor.app/docs/route-explorer) |
| **Scenario Engine** | 运行预置冲突、天气、制裁、关税场景 → 快速做一致 what-if → 应急预案 | Pro v1；固定模板、无滑杆，预留类型未上线，[文档](https://www.worldmonitor.app/docs/scenario-engine) |
| **市场与宏观雷达** | 聚合股指、商品、加密、FRED、央行、能源和预测市场 → 将事件与市场反馈并看 → 宏观/能源 | 基础 Free；增强源需 Key，高级研究 Pro |
| **个性化监控与回放** | 关键词、媒体开关、布局、7 天快照、CSV/JSON/Markdown 导出 → 形成个人值班台 → 长期主题追踪 | 基础本地可用；团队共享不在 Free |
| **六种站点 + 桌面 + 25 语** | 提供 world/tech/finance/commodity/happy/energy 与跨平台桌面端 → 垂直场景开箱 → 跨语言团队 | 站点和桌面标 Stable；桌面 CI 主要发布 full/tech |
| **MCP + REST + CLI + SDK** | 约 40 个情报工具、OpenAPI、CLI、Python/Ruby/Go/JS SDK → Agent 无需抓网页 → Grounding/内部 Bot | 列工具可匿名，调用多需 Pro/API Key；官方 40/41 口径漂移，[MCP](https://www.worldmonitor.app/docs/mcp-overview) |

**易用性与同类差异：** [公共 Demo](https://www.worldmonitor.app/) 无需注册，另有桌面、CLI、OpenAPI、四种 SDK、六个预配置站点和完整方法学文档。相较 Liveuamap，它跨域图层、供应链场景和 Agent 接口更宽，但缺少编辑团队逐条核验；相较 GDELT/ACLED，它更易用，却缺长期历史深度与领域权威。

**限制与关注原因：** “实时”通常是 5–15 分钟且各源不同；陆基 AIS 覆盖不均，Telegram/AI 摘要只是线索；Local AI 不等于世界数据、服务器评分和 MCP 全离线；AGPL 自托管有合规义务。事实：今日 #1、+1,295，最新 [v2.5.23](https://github.com/koala73/worldmonitor/releases/tag/v2.5.23)。推断：公开可玩的高密度地图，加上 MCP、本地 AI 和供应链 Scenario，形成了极强的可视传播组合。

### 15. [Outlines](https://github.com/dottxt-ai/outlines)

**一句话定位：** 面向 Python LLM 应用开发者的结构化生成 SDK，把 Python 类型、Pydantic/JSON Schema、正则或 CFG 作为输出约束，使结果更容易安全交给分类、抽取、函数参数和下游代码。[1.3.2 文档](https://github.com/dottxt-ai/outlines/tree/1.3.2)

**典型场景：** 信息抽取/入库；有限集合分类；函数参数生成；严格格式/DSL；跨本地/云模型的结构化应用。

| Feature | 能做什么 → 用户价值 → 适用场景 | 状态 / 边界 |
|---|---|---|
| **Python 类型契约** | 直接用 int/bool/List/Dict/Union 等约束 → 少写 Schema → 简单结构任务 | 仅部分本地/服务适配器支持 |
| **Pydantic / JSON Schema** | 约束字段、枚举、嵌套对象 → 少畸形 JSON/字段漂移 → 抽取/API payload | 最成熟；Anthropic 不支持，Gemini 有限 |
| **函数签名复用** | 从参数名/类型推导 Schema → 避免双份维护 → 工具参数 | 只产参数，不执行函数/权限/Agent loop |
| **Choice / Enum** | 输出只能来自候选 → 不发明标签 → 分类/路由 | OpenAI/Anthropic/Ollama 等并非都支持 |
| **Regex** | 生成时匹配格式 → 少事后拒绝/重试 → 日期/ID/CSV | 只保证形式，不保证业务正确 |
| **CFG** | 用 Lark grammar 约束递归语言 → 表达 Regex 不足的 DSL → 代码/查询片段 | 兼容面最窄；主要本地推理适配器 |
| **Model / Generator API** | `model(prompt, output_type)` 或固定 Generator，统一 call/stream/batch → 易换模型/复用契约 → 重复任务 | v1 返回 raw string；旧 API 需迁移 |
| **15 个 Provider/模型适配** | 接 Transformers、Ollama、vLLM、OpenAI、Gemini、Anthropic 等 → 本地/云可切换 → 隐私/成本取舍 | 每家结构类型、视觉、流式、异步、批量不等价 |
| **Stream / Async / Batch** | 部分适配器提供流式、异步、批量 → 改善在线/离线吞吐 → API/文档处理 | 非统一保证；partial structured streaming 尚未交付 |
| **Chat / 多模态输入** | 在支持模型上用图片/音频/视频输入并输出结构 → 票据/截图抽取 → 文档数字化 | 多模态来自底层模型，Outlines 负责适配/约束 |
| **Template / Application** | Jinja 模板与输出契约封装成跨模型组件 → 少散落 Prompt 拼接 → 稳定分类器 | 无可视工作流、观测或部署控制台 |

**易用性与同类差异：** `pip install outlines`，Python API 短，有清晰的 Provider 功能矩阵、Cookbook、Template/Application；无 Web UI、完整 CLI 或自带 REST。相较 Instructor，Outlines 强在生成时 JSON/Regex/CFG 约束，弱在 Pydantic 解析、语义校验反馈与自动重试；相较 Guidance，它更简单地映射 Python 类型，弱在条件/循环/工具编排。

**限制与关注原因：** “Guaranteed valid structure”不保证事实、业务语义、引用或函数安全；v1 即使传 Pydantic 也返回字符串，需调用者解析；`max_new_tokens` 截断会破坏结构；Provider 矩阵差异明显；`anyOf/oneOf`、optional/nullable 仍有开放缺陷，partial structured streaming 仍是请求。事实：今日 #20、+65；两天前发布 [1.3.2](https://github.com/dottxt-ai/outlines/releases/tag/1.3.2) 修 Schema/vLLM/文档兼容。推断：Agent 工具调用和抽取/RAG 下游越来越依赖稳定机器可读输出。

## 今天最值得体验的 3 个项目

### 1. [World Monitor](https://github.com/koala73/worldmonitor)

**最吸引人的 Features：** 无需注册即可体验 56 类地图和 500+ feeds；国家风险、跨流关联、Route/Scenario 把新闻变成可操作的研究界面；MCP/API/CLI/四 SDK 又让同一数据面可供 Agent 使用。

**建议体验路径：** 先打开[公共站点](https://www.worldmonitor.app/)，选择一个热点国家核对地图、Brief 和原始来源；再运行 `npx worldmonitor tools` 查看 MCP 工具。不要先付费，也不要把 AI Brief 当作已核实结论。

### 2. [Code Review Graph](https://github.com/tirth8205/code-review-graph)

**最吸引人的 Features：** 本地增量代码图、Blast Radius、受影响测试/流程、最小 Review Context、GitHub Action 和 VS Code/D3 可视化组成了完整的 Coding Agent 上下文层。

**建议体验路径：** 在一个几百到几千文件的真实仓库安装并 `build`，挑 2–3 个已知影响面的 Diff 对拍 callers/tests，再看 token 估算；只有图与源码/LSP 基本一致时，才接入 MCP 或 CI。

### 3. [Wigolo](https://github.com/KnockOutEZ/wigolo)

**最吸引人的 Features：** 一个本地工具面同时提供 Search、Fetch、Crawl、Extract、Cache、Research、Diff 与 Watch；结果带来源、评分与降级信息，并可经 MCP/CLI/REST/SDK 接入现有 Agent。

**建议体验路径：** 先用 `npx wigolo` 对同一技术问题做 Search→Fetch→Cache，再对一个版本支持页试 Diff；中文查询、Remote MCP、ARM64 Docker 或长期 Watch 应先针对开放 Issues 做烟测。

> 若你的首要问题是“本机到底能跑哪个模型”，则 [llmfit](https://github.com/AlexsJones/llmfit) 是最值得替换进入前三的工具；若更关注 CAD/机器人，优先试 [text-to-cad](https://github.com/earthtojake/text-to-cad) 的 Viewer 与一个小型 STEP 任务。

## 值得持续跟踪的新 Feature 方向

1. **Agent Context Control Plane：** 代码关系图、Web 证据缓存、业务数据 MCP、实时情报 API 正逐渐替代“把更多文本塞进 Prompt”；下一步要看证据溯源、权限、时效和跨源冲突如何统一。
2. **专业桌面 Agent 接口：** TradingView MCP 展示了“直接操作用户已有专业工作区”的价值；CAD、IDE、设计、数据分析工具会继续出现类似桥接，但 UI 状态一致性与最小权限将成为核心门槛。
3. **Skill 的发布、评测与权限清单：** 今天多个项目把流程封装成 Skill；真正缺的是版本、宿主兼容矩阵、自动 Eval、所需工具/数据/副作用声明，以及组织级安装治理。
4. **多模型预算与质量联合路由：** OmniRoute 已覆盖配额、成本、fallback 和压缩，llmfit 覆盖硬件适配；下一阶段应把任务成功率、长上下文退化、实测吞吐和数据合规纳入路由，而非只看免费/便宜。
5. **可验证的结构化输出：** Outlines 解决“形式合法”，但字段事实、业务语义、流式局部验证和失败重试仍需更完整闭环；这是 Agent 工具调用进入生产的基础设施方向。
6. **语义层成为 Agent Grounding 产品：** Ontology Playground 说明本体/业务词汇正在从数据建模议题进入 Agent 体验；值得跟踪从静态 Schema、实例图、权限到可审计动作的完整链路。
7. **Local-first 的诚实边界：** 本地缓存、模型、Gateway 与桌面桥接会继续增长；优质项目应明确哪些数据出网、哪些功能需付费/Key、哪些默认监听公网、哪些 License 触发网络分发义务。
