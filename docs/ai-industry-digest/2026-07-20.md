## 今日最重要的趋势

1. **具身模型同时向“更多廉价数据”和“更小本地模型”推进。** Riemann-1.0 尝试把 20 万余小时人类第一视角视频转成机器人动作先验，MiniCPM-Robot 则用 0.9B/1.5B 模型、视觉 Token 压缩与硬件适配降低本地部署门槛。两条路线共同指向：具身智能的竞争不再只看参数，而是看数据规模、历史记忆、控制时延和跨本体泛化。
2. **Agent Harness 正从人工配置层变成可评测、可进化的系统对象。** Self-Harness 不更换底层模型，而是从失败轨迹中挖掘弱点、提出受约束修改，再用 held-in/held-out 回归测试门控采纳。Harness、验证器与运行策略正在成为模型之外的重要性能层。
3. **Coding Agent 的爆发开始直接冲击容量规划与订阅经济。** Kimi K3 上线后暂停 C 端新用户订阅，并计划拆分普通产品与高算力 Kimi Code 权益。停售是明确的需求与容量信号，但收入、留存、推理毛利和长期使用强度仍需后续披露验证。
4. **开放技术产物正在成为发布可信度的分水岭。** MiniCPM-Robot 与 Self-Harness 有代码、权重或论文可核对；Riemann-1.0 的成绩目前主要来自厂商发布和媒体报道，Kimi 商业数据则包含第三方媒体转述。不能把四类证据放在同一置信层级。
5. **本窗口的内容供给高度集中，不能代表全球议题分布。** 四篇入选稿全部来自量子位，且两篇与 WAIC 期间的具身发布相关；Founder Park 与两个英文来源均无新文。这只能说明监测来源在本窗口的发稿结构，不能推断国际行业讨论只聚焦具身智能。

### 【量子位】1.5B开源通用VLA模型，冲进具身智能第一梯队

- 作者：十三；正文署名“金磊 发自上海”
- 发布时间：2026-07-20 11:36:20（Asia/Shanghai）
- [原文链接](https://www.qbitai.com/2026/07/455229.html)
- [MiniCPM-Robot 开源仓库](https://github.com/OpenBMB/MiniCPM-Robot)；[RobotManip 权重](https://huggingface.co/openbmb/MiniCPM-RobotManip)；[RobotTrack 权重](https://huggingface.co/openbmb/MiniCPM-RobotTrack)
- 访问范围：完整正文，无付费墙
- 入选依据：抓取时位列量子位首页最新文章前列；模型、权重和部署代码已经公开，Subagent 核验时仓库可见 103 Stars，形成可见的开发者关注信号。站内没有公开阅读、点赞数据，未作推测。

#### 摘要

文章报道面壁智能发布并开源 MiniCPM-Robot 系列：1.5B 的 RobotManip 面向机械臂通用操作，通过视觉 Token 压缩和流式上下文保留长任务历史；0.9B 的 RobotTrack 面向机器狗目标跟踪，可在宇树 Go2 的本地算力上运行。配套 PhyAI 推理框架负责硬件适配和加速。发布把模型能力、端侧时延、隐私、断网可用性与部署成本放进同一套具身工程叙事，但公开 benchmark 主要仍是项目方报告值。

#### 关键观点与事实

- **【可核验事实】** [OpenBMB/MiniCPM-Robot](https://github.com/OpenBMB/MiniCPM-Robot)以 Apache-2.0 开源，并提供 1.5B RobotManip、0.9B RobotTrack 的代码、模型权重和部署入口。
- **【可核验事实】** 项目表列 RobotManip 的 RMBench 为 53.3，π0.5 为 10.4；同表给出的 H100 BF16 单帧模型前向时延分别为 120ms 与 234ms。这是官方仓库数据，本次未独立复现实验。
- **【可核验事实】** 项目表列 RobotTrack 在 EVT-Bench 三类任务的跟踪率为 89.8%、73.4%、80.4%，并声明在 Go2 本地达到 5+ FPS、约 180ms 端到端时延。
- **【文章陈述／项目方数据】** PhyAI 在 H20 上把 MiniCPM-Robot 推理吞吐从约 10Hz 提高到约 37Hz；提升来自 CUDA Graph 和自定义 Triton 融合算子。

**行业意义：** **【主 Agent 推断】** 具身模型竞争正从单纯扩大参数转向“单位算力能力、历史记忆、硬件适配和本地运行”的综合工程指标。小模型加开放部署链可缩短机器人厂商的验证周期，但跨本体迁移、真机稳定性和长期任务表现仍需第三方复现。

### 【量子位】看了20万小时「人类干活实录」，机器人悟了

- 作者：一水
- 发布时间：2026-07-19 16:32:36（Asia/Shanghai）
- [原文链接](https://www.qbitai.com/2026/07/454592.html)
- 访问范围：完整正文，无付费墙
- 入选依据：抓取时位于量子位首页头图推荐位；WAIC 首发的世界动作模型，关注人类视频规模化训练机器人这一重要路线。文章页没有公开互动数，且尚未发现独立论文、模型卡或官方 benchmark 提交记录。

#### 摘要

文章介绍昆仑万维旗下黎曼动力发布的世界动作模型 Riemann-1.0。模型使用约 23.2 万小时多源数据，其中包括 20 万余小时第一视角人类视频、1.2 万余小时 UMI／外骨骼手套数据和 2 万余小时真机及仿真轨迹，覆盖 41 种机器人本体。其训练路线先从人类视频生成伪动作，再用真实动作校准并以机器人数据强化，试图把廉价、无标签的人类操作视频转成控制能力。文章报告多项基准和真机成绩，但证据目前主要来自发布方。

#### 关键观点与事实

- **【文章陈述】** Riemann-1.0 把视频动态与动作序列放入全因果联合生成框架，并采用“人类视频伪动作预训练—真实动作校准—机器人数据强化”三阶段训练。
- **【文章陈述】** 模型数据覆盖约 23.2 万小时、41 种机器人本体；人类第一视角视频占 20 万余小时，是训练语料的主体。
- **【文章陈述】** RoboCasa-365 成功率为 62.6%，比此前 SOTA 高 8.4 个百分点；加入人类视频后，相关消融成绩由 48.2% 升至 62.6%。
- **【文章陈述】** LIBERO、RoboTwin 2.0 成绩分别为 99.0%、94.3%；四类真机家务任务平均成功率为 85.00%、过程完成度为 94.43%。本次调研未发现可独立核验这些数字的论文或模型卡。
- **【可核验事实】** RoboCasa365 是公开机器人基准，覆盖 365 项任务和约 2500 个厨房环境；这只能核验基准存在，不能独立证明 Riemann-1.0 的成绩。

**行业意义：** **【主 Agent 推断】** 如果结果能够独立复现，大规模人类视频可能成为具身模型的“互联网预训练语料”，缓解遥操作和真机数据昂贵、覆盖不足的问题。当前不应由单次厂商发布直接推导出跨本体泛化已经解决，尤其需要公开数据配方、评测协议和第三方真机测试。

### 【量子位】不换模型，效果提升104%！上海AI Lab让Harness也能自进化了

- 作者：Jay；Self-Harness 团队投稿
- 发布时间：2026-07-19 15:00:47（Asia/Shanghai）
- [原文链接](https://www.qbitai.com/2026/07/454441.html)
- [论文：Self-Harness: Harnesses That Improve Themselves](https://arxiv.org/abs/2606.09498)；[开源仓库](https://github.com/qzzqzzb/Self-Harness)
- 访问范围：完整正文，无付费墙
- 入选依据：论文和代码公开；文章展示该工作获 LangChain 联合创始人 Harrison Chase 转发，并被 Lilian Weng 的自进化 Agent 博文收录，属于可见的专业传播信号。站内没有公开互动数。

#### 摘要

Self-Harness 将 Agent 外层的系统提示、工具规则、验证器和运行控制策略也视为可优化对象。系统先从失败轨迹中归纳模型特有的弱点，再让同一模型提出受约束的 Harness 修改，最后通过 held-in/held-out 回归测试决定是否保留。在模型、工具环境和评测协议固定的情况下，论文报告三个模型后端均有提升。它展示的不是无限自我进化，而是一条“观察失败—提出修改—用外部评测门控”的工程化迭代路径。

#### 关键观点与事实

- **【可核验事实】** 论文提出 Weakness Mining、Harness Proposal、Proposal Validation 三阶段循环，修改对象包括系统提示、工具说明、验证器和轻量运行策略。
- **【可核验事实】** 论文报告 held-out 通过率：MiniMax M2.5 从 40.5% 升至 61.9%，Qwen3.5-35B-A3B 从 23.8% 升至 38.1%，GLM-5 从 42.9% 升至 57.1%。
- **【文章陈述】** 按文章的总体指标口径，三个模型的提升分别为 28%、104%、24%；论文还分别报告 held-in/held-out 相对增幅，使用这些数字时不能混用口径。
- **【可核验事实】** 当前实验限定于 Terminal-Bench-2.0、固定模型后端和 64 个主评测案例，不能直接外推到所有 Agent 工作负载。

**行业意义：** **【主 Agent 推断】** Agent 的实际竞争力越来越取决于“模型 × Harness”的组合。自动回归门控有望降低多模型、多任务的人工调参成本，但也可能对单一 benchmark 过拟合；生产采用还需要跨任务评测、成本约束、安全审查、变更审计和可靠回滚。
