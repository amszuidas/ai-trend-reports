1. **开放模型竞争从“开放权重”升级到“可定制底座与首日生态”。** Inkling 以近 1T 参数、原生多模态、Apache 2.0 和多套推理框架首日支持，展示了模型、许可证、后训练和部署生态协同发布的新范式。

### 【Latent Space】[AINews] Thinky's Inkling: 975B-A41B multimodal, new best American Apache 2.0 open model (with Inkling-Small, 276B-A12B)

- 中文译名：Thinky Inkling：975B-A41B 多模态，美国新晋重要 Apache 2.0 开放模型
- 作者/署名：Latent.Space；公开页未单列个人作者
- 发布时间：2026-07-16 14:18:05（Asia/Shanghai）
- [原文链接](https://www.latent.space/p/ainews-thinkys-inkling-975b-a41b)
- 访问范围：Paid；仅总结公开预览，未绕过付费墙
- 入选依据：本窗口 Latent Space 唯一新增文章，也是 Feed 最新条目；主题为 Thinking Machines Lab 首个完整公开权重基础模型。抓取时，[Hugging Face 技术文章](https://huggingface.co/blog/thinkingmachines-inkling)显示 48 次赞同，[官方模型页](https://huggingface.co/thinkingmachines/Inkling)显示约 599 个赞，构成可见外部关注信号。

#### 摘要

本期 AINews 聚焦 Thinking Machines Lab 首个完整公开权重基础模型 Inkling。文章将其定位为可定制的多模态底座，而非单纯冲榜的 SOTA：模型采用 975B 总参数、41B 激活参数的 MoE，支持文本、图像和音频输入、最高 1M 上下文，并以 Apache 2.0 发布。报道汇总架构、基准和首日推理生态，认为它成为美国开放权重阵营的重要新基线，但在部分 Agent、终端和多模态评测上仍落后于 GLM、Kimi 等模型。“美国最佳开放模型”、训练数据来源与蒸馏纯粹性仍存在口径争议。

#### 关键观点与事实

- **【文章陈述】** Inkling 更强调可定制性、可控推理和广泛能力；“美国最佳开放模型”是文章汇总社区评价后的判断，并非无争议事实。
- **【可核验事实】** 官方模型页标明 Apache 2.0、975B 总参数、41B 激活参数，以及文本、图像和音频输入；Hugging Face 技术文章确认 45 万亿训练 token 和最高 1M 上下文。
- **【可核验事实】** 架构采用相对位置注意力、5:1 的局部/全局混合注意力、短卷积，以及每 token 路由 6 个专家并常驻 2 个共享专家。
- **【文章陈述】** 公开汇总显示 Inkling 在部分 Terminal、Agent 和多模态评测中仍落后于 GLM 5.2、Kimi K2.6 或顶级闭源模型。
- **【可核验事实】** 发布首日已有 Transformers、SGLang、vLLM、llama.cpp 等支持；完整 BF16 权重约需 2TB VRAM，NVFP4 版本约需 600GB。

**行业意义：** **【主 Agent 推断】** Inkling 的战略价值可能大于单项榜单价值：许可证宽松、原生多模态和面向后训练的定位，为企业私有数据、Agent 和行业任务提供了新底座；但高部署门槛会让其近期影响更多经由托管推理、量化版本和 Tinker 等服务释放。

### 【量子位】刚刚，逐际动力放出一段Demo，Figure：睡不着啊

- 作者：田晏林
- 发布时间：2026-07-16 16:04:18（Asia/Shanghai）
- [原文链接](https://www.qbitai.com/2026/07/451084.html)
- 入选依据：位于首页显著位置；涉及刚完成 2 亿美元 Pre-IPO 融资的具身智能公司及新的长程真机演示，文章给出架构、控制频率和对比指标。无公开阅读量或互动数字，演示真实性和评测仍主要由公司及文章陈述支持。

#### 摘要

文章围绕逐际动力全尺寸人形机器人 Oli 的三分钟家庭任务演示，介绍 LimX COSA 0.5“大脑系统”。据报道，Oli 在无远程操作和无剪辑条件下连续完成衣物收纳、玩具整理、搬箱和弯腰拾物。技术路线不押注单一端到端模型，而是拆成 S2 认知决策、S1 可复用技能和 S0 全身运动控制三层，分别以约 1Hz、50Hz 和 1000Hz 运行。文章还给出 LimX WBT 与 SONIC 的运动跟踪对比，并称系统利用专家接管数据和奖励模型实施真机强化学习；与 Figure 的优劣比较仍缺统一第三方测试。

#### 关键观点与事实

- **【文章陈述】** Oli 完成三分钟一镜到底、无后台遥操作的长程家庭任务演示。
- **【文章陈述】** COSA 0.5 按 S2 认知、S1 技能、S0 运控三层解耦，运行频率分别约为 1Hz、50Hz 和 1000Hz。
- **【文章陈述】** S0 使用约千万参数、完全在机器人本体运行的 LimX WBT Transformer 策略。
- **【文章陈述】** 公司提供的对比中，LimX WBT 的 MPJPE 为 12.85 毫米，低于 SONIC 的 13.75 毫米；平均关节角误差为 1.5 度，对比 3.3 度。
- **【可核验事实】** 量子位正文公开展示了演示、架构说明和对比数据，但未提供独立第三方复现实验。

**行业意义：** **【主 Agent 推断】** 该路线代表具身智能从“一个大模型控制全部动作”转向分层系统工程，让低频推理、中频技能和高频控制分别优化；是否优于端到端路线，仍需标准化任务、失败率和长期运行数据验证。

## Agent、记忆与数据治理

### 【量子位】一位失忆患者，揭开了AI记忆的误区

- 作者：闻乐
- 发布时间：2026-07-16 15:34:14（Asia/Shanghai）
- [原文链接](https://www.qbitai.com/2026/07/451049.html)
- 入选依据：首页新稿；议题对应 Agent 长期运行中的核心基础设施问题，涵盖记忆检索、事实存档、动态状态和跨 Agent 共享。页面没有公开互动数据；对 Shadoweave 产品和基准领先的描述缺少详细分数或第三方链接。

#### 摘要

文章从神经科学病例出发，提出“存储信息”和“按当前问题重建回忆”应当分离。它批评仅依靠向量相似度、RAG 或超长上下文保存对话的方案，认为内容相似不等于与当前问题相关。Shadoweave 的 HMS 架构据称会冻结原始历史记录，再依据时间、人物和多条线索重建答案，并通过反馈处理去重、日期校正和事实状态。团队还计划以 Memory Bank 统一管理跨应用、设备、模型和 Agent 的长期状态，并发展 Memory IR、Runtime、SDK 和协议层。文章具有产品介绍色彩，基准领先和跨平台能力仍待外部验证。

#### 关键观点与事实

- **【文章陈述】** 高效记忆不仅需要保存，还需要筛选、遗忘、重建和承认错误。
- **【文章陈述】** HMS 将不可修改的事实存档与面向当前问题的回忆推理解耦。
- **【文章陈述】** HMS 在 LongMemEval 与 LoCoMo 两项测试中领先，但正文未列具体分数或可复现链接。
- **【文章陈述】** Memory Bank 希望让不同模型和 Agent 共享统一长期记忆，并为用户提供来源、时间、置信度和删除控制。
- **【可核验事实】** 文章披露了 Memory IR、Memory Runtime、Memory SDK 和新 Benchmark 的路线，但公开材料不足以核验完成度。

**行业意义：** **【主 Agent 推断】** 如果 Agent 从短会话工具发展为持续数月运行的协作者，记忆管理可能成为独立于模型推理的基础设施层；机会伴随数据所有权、错误记忆传播和跨应用权限治理风险。

## 方法与口径

- 窗口从自动化提供的上次成功运行时间继续覆盖，不按自然日或 13:00 强行截断。
- 四个来源分别由四个独立 Subagent 调研；每个 Subagent 只负责一个来源，并实际打开正文、公开预览或官方活动页。
- 主 Agent 对模型规格、开源仓库、入口页和公开热度信号进行了交叉核验，并与上一份日报去重。
- 入选依据综合官网或首页位置、发布时间、公开互动、引用传播、事件重要性和信息密度；没有互动数字时不作杜撰。
- 付费文章只总结公开预览，未绕过付费墙。
- 厂商供稿、公司 Demo、内部基准、活动主办方数字和社区评价均按证据性质明确归因。
- 全程未使用机器之心。
