# 2026-07-21 中国早期 AI 初创求职雷达

- 调研完成：2026-07-21（Asia/Shanghai）
- 本期类型：首次基线，四家公司聚焦版
- 目标画像：Agent Harness、Agent Runtime、工具执行、沙箱、编排、上下文、权限与可观测性工程师
- 纳入门槛：公司主要团队位于中国大陆；种子、天使、A/B 轮；公开口径员工总数不超过 100 人
- 重点公司：词元无限、BetterYeah AI、VUI Labs、心识宇宙

> 本期复用此前市场扫描，不重新铺开长名单。融资和员工数采用“独立报道／公司公告 + 招聘平台”交叉核验；客户数、收入预测、性能、使用量和 ROI 若只有公司口径，均明确标为自报，不把它们当作已审计事实。招聘平台的轮次和人数常有滞后，应在面试中二次确认。

## 先看结论

四家公司都值得进入面试漏斗，但“值得聊”和“值得接 Offer”不是一回事。对 Agent Harness / Runtime 工程师，我会采用两套排序：

1. **纯技术匹配度：词元无限 > BetterYeah AI > VUI Labs > 心识宇宙。** 词元无限最直接地把 Harness、工具执行、容器沙箱、长任务编排和对抗式验证写进产品与技术资产；BetterYeah 的强项是已经面向企业生产环境的多租户 Agent 平台。
2. **风险调整后的加入优先级：BetterYeah AI > 词元无限 > VUI Labs > 心识宇宙。** BetterYeah 的融资、产品文档、招聘面和商业落地证据更完整；词元无限技术方向更贴，但公司极年轻，商业数字主要来自公司披露。

| 优先级 | 公司 | 当前判断 | 最适合争取的工作所有权 | 下一步 |
|---:|---|---|---|---|
| 1 | **BetterYeah AI** | 产品和商业成熟度最佳；需确认是做平台内核还是客户交付 | 多租户 Runtime、工作流执行、插件/MCP、权限、调试与可观测性 | 优先投核心平台岗，要求和 CTO／平台负责人聊系统边界 |
| 2 | **词元无限** | 技术画像最贴近 Harness / Runtime；高上行、高波动 | 工具执行、容器沙箱、上下文、长任务调度、测试—修复闭环 | 直接联系创始团队，先核实收入质量和平台岗占比 |
| 3 | **VUI Labs** | 实时系统问题很硬；公开产品目前更像 AI 听写楔子 | 流式会话 Runtime、打断、低延迟调度、API 平台与 SRE | 若愿意转向语音实时系统，定向争取平台／后端岗 |
| 4 | **心识宇宙** | 记忆和个性化技术有吸引力；产品主线与当前高级岗位不够清晰 | 长期记忆、个性化模型服务、本地 Runtime、容器与 A2A | 只通过创始团队或技术负责人做探索性沟通，不盲投 |

## 1. 词元无限（TokensInfinity）：技术匹配度第一

### 现在是什么状态

- [公司官网](https://www.tokfinity.com/company)称团队于 2025 年 7 月成立，创始人杨平曾任字节跳动 AI 技术负责人，联合创始人兼 CTO 王玮来自清华姚班；公司聚焦 AI 原生研发 Agent。
- 2026 年中，公司完成数千万元天使+轮，投资方包括华控基金、水木创投和厦金创新。[智源社区转载的 Founder Park 报道](https://hub.baai.ac.cn/view/55177)同时披露了客户、订单和收入预测，但这些经营数字来自公司采访口径，并非审计数据。
- [BOSS 直聘公司页](https://www.zhipin.com/zhaopin/47c0dbcb85401b451Hx609q0/)标注天使轮、20—99 人；官网公开招聘邮箱为 `hr@tokfinity.com`，并明确欢迎 Agent／系统架构人才。
- 产品包括 [InfCode](https://www.tokfinity.com/infcode)、InfTest 和项目理解能力，覆盖 IDE、CLI、企业知识库、权限、审计与私有化部署。公开技术资产还包括 [InfCode GitHub 仓库](https://github.com/Tokfinity/InfCode)和 [InfCode 论文](https://arxiv.org/abs/2511.16004)。

### 为什么和 Agent Harness / Runtime 高度匹配

InfCode 不是简单的代码补全壳。其公开架构把代码修改 Agent、测试 Agent、候选选择器、容器化仓库环境、文件编辑／搜索／Shell 工具和模型接口放在同一执行闭环里；论文报告的 SWE-bench Verified 79.4% 是作者自报结果，仍需看可复现配置，但技术问题本身与 Harness / Runtime 高度重合：

- 每个任务的隔离环境、镜像构建、工具权限和资源回收；
- 长上下文与仓库级状态管理，MCP、Rules、Skills 的组合；
- 多 Agent 调度、测试—修复对抗、候选轨迹选择与失败恢复；
- 企业私有部署、知识库、角色权限、审计和成本／效率观测。

如果岗位能拥有上述内核，而不是只做客户插件和项目部署，这四家公司里它最接近你的现有积累。

### 商业信号与需要打折的数字

官网展示金融、供应链、轨交和互联网方向客户，并给出神州信息项目“综合效能提升 39%、成本下降 20%、代码采纳率 88%”等案例；Founder Park 报道还转述“十余家头部客户、多个千万级战略订单、年收入预计过亿元”等说法。它们能证明公司已经开始企业销售，却不能证明回款、续费、毛利和标准化产品收入。尤其“预计收入”不是已实现收入。

### 面试前必须问清

1. 最近 12 个月已确认收入、实际回款、软件订阅／License 与定制交付各占多少？所谓千万级订单按几年确认？
2. 生产任务成功率如何定义？按任务类型拆分后的成功率、人工接管率、P95 时延、单任务 Token／算力成本是多少？
3. Agent 架构岗对核心 Runtime 的代码所有权有多大？一季度内平台研发、客户集成和现场支持各占多少时间？
4. 沙箱的隔离边界、密钥注入、网络出口、制品留存和审计模型是什么？线上出现工具误执行时如何止损和回放？
5. InfCode 论文／开源仓库与商业产品共用多少代码？Benchmark 的模型、采样次数、并发和失败重试预算能否复现？

### 判断

**建议进入第一批面试。** 前提是拿到“核心 Agent 平台”而非“客户项目高级实施”的承诺，并用回款、续费和产品收入结构校正融资稿中的高增长叙事。

## 2. BetterYeah AI（斑头雁）：风险调整后第一

### 现在是什么状态

- BetterYeah 成立于 2021 年。[公司发布页](https://www.betteryeah.com/report)披露其在 2025 年完成超过 1 亿元人民币 B 轮融资，由阿里云领投，并持续发布同年的产品更新；招聘平台的轮次标注与之相符。
- [BOSS 直聘公司页](https://m.zhipin.com/companys/4691e1e0b57649701HB-3tm0FlI~.html)标注 B 轮、20—99 人，抓取时显示 14 个岗位，包括 AI 应用开发、Agent 应用开发、AI Agent 高级研发、Python 和后端架构。
- [官方文档](https://ai-docs.betteryeah.com/)可核验的产品面较完整：Agent、知识库、插件、自定义插件、API／SDK、工作流节点、JavaScript／Python 执行、环境管理、调试日志、数据库、钉钉集成，以及 MCP 服务的创建与调用。更新日志延续到 2026 年，说明并非只剩营销落地页。

### 为什么值得优先聊

BetterYeah 的优势不是某个单 Agent Demo，而是企业 Agent 平台已经暴露出生产 Runtime 必须处理的控制面：多租户、权限、环境隔离、版本和发布、插件／MCP、日志与调试、知识和数据连接、API 嵌入、私有化部署。对 Harness / Runtime 工程师，这意味着有机会直接面对真实企业流量、复杂集成和稳定性约束，而不是只追离线 Benchmark。

公司自报已服务近 10 万个企业团队、累计数十万 Agent 和数十亿次任务，并列出联想、百丽、科沃斯、添可、苏泊尔等客户；[官网案例](https://www.betteryeah.com/)给出响应时间、培训成本和效率提升数字。这些信号比纯 Demo 强，但仍属于公司口径，不能替代 ARR、净收入留存和客户访谈。

### 最大风险：平台研发还是“高配实施”

企业 Agent 平台很容易把最强工程师吸入售前 PoC、客户数据接入、私有化环境排障和一次性工作流定制。公开岗位名中的“应用开发”也说明这一风险是真实的。对你而言，关键不是公司有没有客户，而是岗位是否拥有跨客户复用的 Runtime 内核。

### 面试前必须问清

1. 年度经常性收入、续约率、净收入留存、毛利率分别是多少？订阅、模型转售、实施和定制开发的收入占比如何？
2. 过去两个季度研发工时中，标准平台、客户定制、私有化交付和售前 PoC 各占多少？哪些需求会被拒绝进入主干？
3. 工作流／Agent Runtime 的并发规模、P95／P99 时延、任务失败率、重试放大、单任务成本和最长运行时长是多少？
4. JavaScript／Python 节点、插件和 MCP 的沙箱边界是什么？网络、文件、密钥、CPU／内存、超时与租户隔离如何实现？
5. 这个岗位是否直接负责调度器、状态机、权限、可观测性或执行沙箱？On-call 和客户现场支持的频率是多少？

### 判断

**最适合作为风险调整后的首选。** 如果能确认加入的是平台 Runtime 团队，并且标准化软件收入和续费健康，它比词元无限更可能同时提供技术深度与可持续业务；若岗位主要跟项目走，则优先级应下调。

## 3. VUI Labs（宇生月伴）：实时语音 Runtime 选项

### 现在是什么状态

- [新浪财经转载报道](https://finance.sina.com.cn/tech/roll/2026-02-28/doc-inhpiryq3062221.shtml)称 VUI Labs 于 2026 年 2 月完成数千万元天使+轮，由同创伟业领投，靖亚资本、小苗朗程继续加注；半年累计融资接近 1 亿元。
- [Bonjour 团队页](https://bonjour.bio/jobs-mapping/team/vui-labs)显示团队 10—50 人、位于上海／杭州；[BOSS 直聘页](https://www.zhipin.com/zhaopin/5c269b82256a9c511nVz3dm4/)则标注 A 轮、20—99 人。两者虽都满足本报告门槛，但轮次和人数口径不一致，必须向公司核实。
- 当前最容易独立看到的产品是 [SaySo](https://www.sayso.ai/)：面向 Mac／Windows 和各种输入框的 AI 听写，提供去口头禅、自动整理、改写、翻译和个人词典。它是一个清晰的语音入口，但公开产品证据目前更接近“AI dictation”，还不足以单独证明完整自治 Voice Agent 平台已经规模化。
- 融资报道和公司采访还披露 Luna 系列模型、端到端延迟、TTS 延迟、实时翻译、规划／工具／记忆／编排框架及使用量数据；这些性能和运营指标主要为公司自报，应在技术面中要求压测与线上指标。

### 为什么仍值得聊

语音 Agent 把普通 Runtime 的问题变得更苛刻：双工流式输入输出、端点检测、打断／抢话、会话状态、工具调用期间的填充与恢复、跨地域抖动、端云切分，以及毫秒级延迟和每分钟成本。若 VUI Labs 真正在建设统一 Voice Agent API 平台，这类系统问题对 Runtime 工程师很有含金量。

不过，抓取时能稳定核验的公开职位更偏 API 测试／自动化；核心平台高级岗的即时可用性不如前两家公司明确。更适合直接联系技术负责人，先确认岗位再投入完整面试流程。

### 面试前必须问清

1. SaySo 是现金流产品、数据入口还是 Voice Agent 平台的分发楔子？未来 12 个月收入和研发资源主要投向哪条线？
2. 真实线上对话的 P50／P95 首响应和端到端延迟、打断成功率、并发会话数、断线恢复率、每分钟推理成本分别是多少？
3. ASR、TTS、语义模型和对话编排中哪些自研、哪些依赖第三方？基础模型降价或原生支持实时语音后，公司的不可替代层是什么？
4. 平台岗是否拥有流式会话 Runtime、调度、状态、观测和 API 网关，还是主要做模型服务封装与业务后端？
5. 付费用户／企业客户、周留存、月留存、使用分钟数和毛利如何？融资稿中的使用量能否按付费与免费拆开？

### 判断

**适合对实时系统和语音交互有明确兴趣的人。** 技术难度可能很高，但需要先证实公司不是只有听写产品和模型指标，而确实有可让你长期拥有的 Voice Agent Runtime。

## 4. 心识宇宙（Mindverse）：技术有深度，组织与主线待核

### 现在是什么状态

- [新浪财经转载报道](https://finance.sina.com.cn/jjxw/2026-01-27/doc-inhithnc8324076.shtml)称公司于 2026 年 1 月完成 Pre-A 轮，由蚂蚁集团领投、HSG 等参与，累计融资超过 2,000 万美元。
- [BOSS 直聘公司页](https://m.zhipin.com/companys/ca97d9037cad3e001HF43tq4FlY~.html)仍标注天使轮、20—99 人，法律实体成立于 2022 年；独立融资报道则把团队起点写为 2023 年 10 月。轮次显然滞后，“公司成立”与“现团队成立”也不是同一口径。
- [Second Me 开源仓库](https://github.com/mindverse/Second-Me)抓取时约 1.55 万 Stars、1,200 Forks，采用 Apache-2.0，展示了层级记忆、个性化、本地训练／部署、角色与应用、GraphRAG 等能力；[Second Me 论文](https://arxiv.org/abs/2503.08102)为其长期记忆和本地个性化提供了可阅读的技术资产。
- 2025—2026 年公开重心逐渐转向 Macaron AI、Personal Intelligence、每用户个性化模型和生成式 UI；[投资界 2026 年报道](https://news.pedaily.cn/20260602/131230.shtml)使用了“心洲科技”名称，并转述用户与 Mini-App 数据。[Macaron-A2UI 论文](https://arxiv.org/abs/2605.24830)进一步说明生成式界面方向。品牌、主体、IP 和团队关系需要在面试中明确。
- 当前招聘页稳定可见的岗位以 AI Native 全栈实习生等为主，高级核心 Runtime 岗公开供给偏少。

### 为什么对 Runtime 工程师仍有吸引力

Second Me 的问题空间与普通工作流 Agent 不同：跨会话长期记忆、个体数据治理、本地与云端部署、每用户模型生命周期、个性化推理、容器快速拉起、Agent-to-Agent 身份与通信。若新产品仍继承这些内核，工作会涉及状态量大、生命周期长、隐私要求高的 Runtime，而不是只做 RAG API 拼装。

但公开叙事从 Me.bot、Second Me 演进到 Macaron AI／Personal Intelligence，说明公司可能仍在寻找最强产品形态。融资报道中的生成速度、容器启动、成功率、用户和 Mini-App 数字均来自公司口径，而且不同报道对 Mini-App 数量并不完全一致，不宜强行合并。

### 面试前必须问清

1. 当前唯一旗舰产品和未来 12 个月北极星指标是什么？Second Me、Macaron AI 与 Me.bot 各自还有多少研发和商业资源？
2. “心识宇宙”与“心洲科技”的法律主体、员工合同、期权池、知识产权和融资主体分别是什么？
3. 30／90 日留存、付费转化、每活跃用户推理／训练成本、个性化模型复用率是多少？免费用户是否让单位经济失真？
4. 本地记忆、云端记忆、用户模型和第三方工具之间如何做权限、删除、导出、隔离与可审计性？
5. 是否存在高级 Runtime 正式岗？该岗拥有记忆服务、容器调度、模型服务或 A2A 协议中的哪一层，而不是泛全栈补位？

### 判断

**适合作为高技术兴趣、低确定性的探索项。** 先通过创始人、技术负责人或投资人转介绍确认产品主线和正式岗位，再决定是否投入；仅凭开源热度和融资新闻不足以接 Offer。

## 面试选择策略

### 建议的实际顺序

1. **同时启动 BetterYeah AI 与词元无限。** 前者用来检验“成熟企业平台是否真有内核所有权”，后者用来检验“高匹配早期团队是否已有健康商业基础”。两条流程互为参照。
2. **把 VUI Labs 当作系统方向选择，而不是备胎。** 如果你愿意把专长迁移到实时语音、流式会话和低延迟服务，它的优先级可升至第二；若仍想深耕通用 Agent Harness，则维持第三。
3. **对心识宇宙采用定向沟通。** 没有高级 Runtime 正式岗位、产品主线或期权主体的明确答复前，不进入多轮作业式面试。

### Offer 前统一索取的证据

- 最近 12 个月实际回款、ARR／可重复收入、续约率、毛利和现金 runway，而不是融资稿中的预测；
- 组织图、直属负责人、岗位前 90 天目标、核心仓库代码所有权和平台／交付工时比例；
- 真实生产指标：成功率定义、人工接管、P95／P99、成本、最大并发、事故与恢复；
- 期权授予主体、完全稀释比例、行权价、归属、离职行权窗口和下一轮稀释情景；
- 至少与一位同组工程师、一位产品／交付负责人交叉核验创始人的说法。

## 本期结论

- **最值得先投：BetterYeah AI、词元无限。** 两者分别代表“已有企业生产面的平台”和“最贴近 Harness 内核的新团队”。
- **最值得条件式加入：VUI Labs。** 条件是你认同语音实时系统方向，并能拿到 Voice Agent Runtime 的明确所有权。
- **最值得继续观察：心识宇宙。** 技术资产不弱，但当前产品中心、主体关系和高级招聘入口都需要更多一手确认。
- 下一期应重点追踪：四家公司新增核心岗位、融资后团队扩张、标准化收入／续约信号、公开技术资产更新，以及本期列出的口径冲突是否得到澄清。
